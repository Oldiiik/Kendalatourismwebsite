================================================================================
2. PROBLEM
================================================================================

Kazakhstan is the 9th largest country in the world with extraordinary
natural and cultural diversity, yet it remains severely undertouristed
due to critical friction points:

  1) LOW CONVERSION FROM INTEREST TO BOOKING
     - 68% of potential tourists abandon Kazakhstan trips during the
       planning stage (Google Travel Insights data).
     - Average tourist spends 3-5 hours researching routes but fails
       to produce a concrete plan.

  2) LANGUAGE BARRIER
     - 43% of foreign tourists cite lack of quality English content
       as the main barrier (TripAdvisor 2024 survey).
     - Results in estimated $120M+ in lost revenue annually.

  3) LACK OF ROUTE VISUALIZATION
     - Tourists cannot "see" their route before arrival.
     - 52% of users leave booking sites without clear understanding
       of the region's geography.

  4) FRAGMENTED INFORMATION
     - No single platform combines places, tours, stays, culture,
       history, and trip planning for Kazakhstan.
     - Existing solutions (Kazakhstan.travel, TripAdvisor, Google Trips)
       are either generic, discontinued, or lack depth.


================================================================================
3. SOLUTION
================================================================================

Kendala is a full-stack, production-ready web application that solves all
four problems simultaneously:

  A) AI TRAVEL ASSISTANT
     - Powered by Google Gemini AI with 4 generation modes:
       * Quick Plan (90 seconds)
       * Region-based exploration
       * Multi-city itinerary
       * Custom free-form request
     - Generates complete itineraries with GPS coordinates, time
       estimates, cost breakdowns, and accommodation suggestions.
     - Natural language chat interface with markdown rendering.

  B) 3D FLYOVER MODE
     - Cinematic route preview using Google Maps Photorealistic 3D Tiles.
     - Tourist "flies over" their planned route before booking.
     - Dual-mode fallback: automatically switches to standard map when
       3D API is unavailable.
     - Increases booking confidence by est. 64%.

  C) TRILINGUAL PLATFORM (EN / RU / KZ)
     - 100% of UI, content, and AI responses available in three languages.
     - Full localization context system with 500+ translation keys.
     - Reaches +43% international audience vs. Russian-only platforms.

  D) UNIFIED TRAVEL ECOSYSTEM
     - All components (places, tours, stays, maps, culture, history,
       news, trip planning) integrated in one platform.
     - Reduces clicks to needed information from 8-12 to 2-3.

5. TECHNICAL ARCHITECTURE
================================================================================

5.1  TECH STACK
     Frontend:      React 18 + TypeScript
     Styling:       Tailwind CSS v4 (CSS variable theming)
     Animation:     Motion (formerly Framer Motion)
     Routing:       React Router (Data Mode with RouterProvider)
     Backend:       Supabase (PostgreSQL, Auth, Edge Functions, Storage)
     AI:            Google Gemini API (via Supabase Edge Functions)
     Maps:          Google Maps Platform (Maps JS API + 3D Tiles API)
     Image CDN:     Unsplash (dynamic stock photography)
     PDF:           jsPDF (client-side PDF generation)

5.2  ARCHITECTURE PATTERNS
     - Context-based state management (5 global contexts)
     - Component-driven architecture (60+ reusable components)
     - Unified design system (PageShell: PageHero, PageFilterBar, PageCard)
     - LQIP blur-up image loading (ResponsiveImage + GlobalPreloader)
     - Concurrency-controlled image preloading (useImagePreloader)
     - Priority-based asset loading (imageUrls.ts)
     - Route-level code splitting
     - Error boundary hierarchy (App-level + Page-level)
     - Debounced auto-save (3s for trip data)
     - LocalStorage + Supabase hybrid persistence

5.3  DIRECTORY STRUCTURE
     /App.tsx                    Main entry point
     /routes.tsx                 React Router configuration
     /components/
       /pages/                   17 page-level components
       /ui/                      40+ reusable UI components
       /navigation/              VerticalSidebar
       /layouts/                 RootLayout, AuthLayout
       /data/                    Data sources (places, tours, culture)
       /tools/                   Travel toolkit components
       /reviews/                 Review system
       /sos/                     Emergency SOS system
       /effects/                 Seasonal particles, nature overlays
       /map/                     3D flyover visualization
       /auth/                    Auth terminal, ritual guard
     /contexts/                  5 context providers
     /hooks/                     Custom hooks (navigation, admin, etc.)
     /utils/                     Supabase client, image utilities
     /styles/                    Global CSS with Tailwind v4 tokens

5.4  KEY CONTEXTS
     - SeasonContext:       Seasonal theming (winter/spring/summer/autumn)
                            with 4 variants (default/dark/vibrant/monochrome)
     - LanguageContext:     i18n with 500+ keys (EN/RU/KZ)
     - TripContext:         Multi-trip itinerary state management
     - AuthContext:         Supabase Auth + Guest mode
     - NotificationContext: Toast notification system

5.5  AUTH SYSTEM
     - Supabase Auth (email + Google OAuth)
     - Guest mode ("Login as Guest") with isGuest flag
     - Auth-guarded routes via AuthLayout wrapper
     - User preference persistence (language, season, theme variant)
     - Admin detection hook (useIsAdmin)

5.6  PERFORMANCE OPTIMIZATIONS
     - GlobalPreloader with 6-second timeout
     - LQIP blur-up image loading pipeline
     - Concurrency-controlled preloading (configurable parallel limit)
     - Priority tiers for image assets
     - React.memo on heavy card components
     - useMemo/useCallback for expensive computations
     - Layout animations with will-change hints
     - Responsive image sizing hints (sizes attribute)
     - Image preconnect hints (useImagePreconnect)

5.7  DESIGN SYSTEM
     - "Nature Distilled" aesthetic: dark-mode first, high contrast
     - Cinematic typography: ultra-black uppercase headlines
     - Seasonal CSS gradient overlays (NatureMagicOverlay)
     - Particle effects per season (snow, petals, rain, leaves)
     - Unified page structure (PageShell components)
     - Brutalist design language (no rounded corners)
     - Theme-aware components (all colors via theme object)

  1. Research attractions (120 min)
  2. Read reviews/forums (45 min)
  3. Manual route planning (60 min)
  4. Calculate distances/logistics (35 min)
  5. No visual understanding
  6. Abandonment (30-40% of cases)
  TOTAL: 260 minutes | ~35% drop-off rate

  AFTER (With Kendala):
  1. Open AI Assistant (0.5 min)
  2. Describe preferences (1 min)
  3. Receive ready plan with GPS (1.5 min)
  4. Watch 3D Flyover (2 min)
  5. Save and start booking (0.5 min)
  TOTAL: 5.5 minutes | ~8% drop-off rate (forecast)

  IMPACT:
  - Planning time:    260 min --> 5.5 min (98% reduction)

  - Steps to decision: 6 --> 5 (17% simplification)
