# 📖 Руководство по составлению промптов для AI-ассистента

## 🎯 Формула эффективного промпта

```
[КОНТЕКСТ] + [ЗАДАЧА] + [ДЕТАЛИ] + [ОГРАНИЧЕНИЯ/ПРЕДПОЧТЕНИЯ]
```

---

## 📐 Структура идеального промпта

### 1. **КОНТЕКСТ** (Background)
Что уже сделано? В какой части проекта работаем?

**Шаблон:**
```
Я работаю над [название фичи/компонента] в приложении Kendala.
Текущее состояние: [краткое описание того, что уже реализовано].
```

**Пример:**
```
Я работаю над страницей Trip Planner. Сейчас у меня есть базовая форма 
с выбором дат и кнопкой "Создать маршрут", но она не сохраняет данные.
```

---

### 2. **ЗАДАЧА** (What to do)
Что именно нужно сделать? Одна четкая цель.

**Шаблон:**
```
Нужно [глагол действия] + [объект] + [результат].
```

**Примеры:**
- ✅ "Добавить сохранение маршрутов в Supabase с привязкой к пользователю"
- ✅ "Создать анимацию появления карточек туров при скролле"
- ✅ "Исправить баг с переводом заголовков в мобильной версии"
- ❌ "Сделать лучше" (слишком размыто)
- ❌ "Улучшить дизайн" (нет конкретики)

---

### 3. **ДЕТАЛИ** (Specifications)
Как именно должно работать? Какие элементы включить?

**Шаблон для UI:**
```
Дизайн:
- Стиль: [минималистичный/кинематографичный/премиальный]
- Цвета: [использовать из globals.css / указать конкретные]
- Анимации: [плавные fade-in / слайды / без анимаций]

Функционал:
- При клике на [элемент] должно [действие]
- Данные загружать из [источник]
- Отображать [какие поля]
```

**Пример:**
```
Создать карточку тура с:
- Изображением (из Unsplash, запрос "kyrgyzstan mountains")
- Заголовком тура
- Длительностью и ценой
- Кнопкой "Узнать больше"
- При наведении — плавное увеличение на 5% и тень
- Использовать премиальный стиль из нашего globals.css
```

**Шаблон для Backend:**
```
API эндпоинт:
- Метод: [GET/POST/PUT/DELETE]
- Путь: /make-server-3ab99f71/[название]
- Принимает: [параметры]
- Возвращает: [структура данных]
- Авторизация: [требуется/не требуется]
```

---

### 4. **ОГРАНИЧЕНИЯ/ПРЕДПОЧТЕНИЯ** (Constraints)
Что НЕ делать? Какие файлы НЕ трогать?

**Шаблон:**
```
Важно:
- НЕ изменять [файл/компонент]
- Использовать ТОЛЬКО [библиотека/подход]
- Сохранить совместимость с [существующий функционал]
```

**Пример:**
```
Важно:
- НЕ трогать GlobalPreloader.tsx
- Использовать существующую локализацию из data_localized.ts
- Анимации должны работать на мобильных устройствах (60 FPS)
```

## 🏆 Примеры отличных промптов

### Пример 1: Новая фича
```
**Контекст:**
В приложении Kendala есть Trip Planner, где пользователи создают маршруты.

**Задача:**
Добавить возможность сохранять созданные маршруты и просматривать их 
в разделе "Мои путешествия".

**Детали:**
1. Создать новую страницу /my-trips с галереей сохраненных маршрутов
2. Каждая карточка содержит:
   - Миниатюру карты маршрута (можно placeholder из Unsplash)
   - Название поездки
   - Даты
   - Количество дней
   - Кнопки "Редактировать" и "Удалить"
3. Данные хранить в Supabase (kv_store с префиксом user_{userId}_trips)
4. При клике на карточку — открывать детальный вид маршрута

**Ограничения:**
- Использовать существующую систему авторизации
- Дизайн в стиле остальных страниц (кинематографичный, премиум)
- Добавить эту страницу в навигацию
```

---

### Пример 2: Исправление бага
```
**Контекст:**
На странице AIAssistantPage при генерации маршрута анимация загрузки 
показывает английский текст, хотя выбран русский язык.

**Задача:**
Исправить баг с переводом текста в компоненте LoadingAnimation.

**Детали:**
- Текст "Crafting your journey..." должен переводиться в "Создаём ваш маршрут..."
- Использовать существующую систему локализации (data_localized.ts)
- Проверить, что переводы работают для всех поддерживаемых языков

**Ограничения:**
- НЕ менять логику анимации, только текст
- Сохранить все существующие стили
```

---

### Пример 3: Оптимизация
```
**Контекст:**
На главной странице загружается много изображений, что замедляет первую загрузку.

**Задача:**
Оптимизировать загрузку изображений с использованием lazy loading.

**Детали:**
- Применить ленивую загрузку для всех изображений ниже первого экрана
- Добавить placeholder'ы с blur-эффектом
- Использовать Intersection Observer API
- Приоритет: hero-изображение грузится сразу, остальные — по мере скролла

**Ограничения:**
- Не ломать существующий GlobalPreloader
- Сохранить плавность анимаций
```

## ❌ Антипаттерны (чего избегать)

### 1. Слишком общие запросы
❌ "Улучши дизайн"
✅ "Увеличь размер заголовков на 20% и добавь золотистый gradient"

### 2. Много задач в одном промпте
❌ "Создай страницу профиля, добавь аутентификацию, исправь баг с датами, 
    оптимизируй изображения и сделай темную тему"
✅ Разбить на отдельные промпты для каждой задачи

### 3. Отсутствие контекста
❌ "Добавь кнопку"
✅ "На странице Trip Planner добавь кнопку 'Сохранить черновик' 
    под формой выбора дат"

### 4. Противоречивые требования
❌ "Сделай минималистичный дизайн с большим количеством анимаций и деталей"
✅ "Сделай минималистичный дизайн с 1-2 тонкими анимациями появления"

## 🎨 Специфика для проекта Kendala

### Стилистические термины
Используйте эти термины для описания желаемого стиля:

- **Кинематографичный**: широкие изображения, темные оверлеи, драматичное освещение
- **Премиальный**: золотистые акценты, большие отступы, элегантные шрифты
- **Кочевнический**: этнические паттерны, натуральные цвета, текстуры
- **Атмосферный**: размытые фоны, градиенты, плавные переходы

### Технические требования
Всегда учитывайте:
- ✅ Поддержка русского и английского языков
- ✅ Респонсивность (мобильные устройства)
- ✅ Работа с Supabase backend
- ✅ Использование существующей системы локализации
- ✅ Совместимость с GlobalPreloader

## 📝 Шаблоны для быстрого старта

### Шаблон: "Создать новую страницу"
```
Создай новую страницу [название] по пути /[route].

Страница должна содержать:
- [элемент 1]
- [элемент 2]
- [элемент 3]

Дизайн: [кинематографичный/премиальный/минималистичный]
Данные: [откуда брать / mock данные]

Добавить страницу в навигацию: [да/нет]
```

### Шаблон: "Исправить баг"
```
На странице [название] происходит [описание проблемы].

Ожидаемое поведение: [как должно работать]
Текущее поведение: [как работает сейчас]

Файл для проверки: [путь к файлу, если известен]
```

### Шаблон: "Добавить фичу к существующему"
```
На странице [название] добавь возможность [действие].

Триггер: [при клике на / при загрузке / при скролле]
Действие: [что должно происходить]
UI: [как визуализировать]

Не изменять: [список компонентов/функций]
```

### Шаблон: "Оптимизация/Рефакторинг"
```
В компоненте [название] нужно оптимизировать [что].

Проблема: [медленно грузится / повторяющийся код / сложно читать]
Цель: [ускорить на X% / упростить / улучшить читаемость]

Сохранить: [функционал, который должен остаться без изменений]
```

## 🚀 Продвинутые техники

### 1. Поэтапные запросы
Для сложных задач разбивайте на этапы:

```
Этап 1: Создай базовую структуру компонента TripCard
Этап 2: Добавь интеграцию с backend для загрузки данных
Этап 3: Добавь анимации и полировку UI
```

### 2. Ссылки на существующее
```
Создай компонент как у [существующий компонент], но с [отличие].
Пример: "Создай SeasonSelector как TripPlanner, но для выбора времени года"
```

### 3. Приоритеты
```
Критично: [обязательный функционал]
Желательно: [nice-to-have фичи]
Можно пропустить: [что можно добавить позже]
```

## 💡 Советы профессионалов

1. **Начинайте с малого**: лучше 3 маленьких четких промпта, чем 1 огромный запутанный
2. **Используйте примеры**: "как на сайте [пример]" или "похоже на [компонент]"
3. **Указывайте приоритеты**: что важнее — скорость работы или красота анимаций?
4. **Проверяйте локализацию**: всегда уточняйте, нужны ли переводы
5. **Думайте о мобильных**: упоминайте адаптивность, если она важна
